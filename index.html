    🚀 TFR (Terminal File Relay) | 流盲终端

    Rust 自研跨网高速加密流转工具，具备终端投射、无痕文件传输、无端口跨网直连能力； 内存透传，用完即焚。

    💻 在目标终端执行以下指令，开启“瞬传”神技：

    eval "$(curl -s https://tfr.sh/t/)" 查看源码

    直接将当前 Session 链接发送给好友。好友打开后，即可在‘好友文件清单’中查收文件，或协助好友运维终端（远程 Shell）。

    https://tfr.sh.cn/t//

    🔒 流盲特性：Session 传输完成后即销毁，中继端无法解密文件内容（密钥仅在收发双方内存中生成）。 开源可审计

    🛠️ 指令速查 (CLI Usage)：

    tfr # 无参数：默认执行拉取我的文件清单选中的文件
    tfr -n # 查看或安装无人值守节点
    tfr [file/dir] # 执行 HTTP PUT 上传 (目录自动压缩) | 小文本支持在线预览与编辑
    tfr [file] -c # 终端续传：从断点恢复下载 (配合发送端 tfr [file] -p)
    💡 提示：浏览器的文件清单支持自动断点续传，无需额外指令。
    echo "data" | tfr [?filename] # 管道模式：透传 STDIN 流 (默认存为 stdin.log)
    tfr -t [?container] [?sh] [?Friend_URL] # 终端投射：将容器/本地 Shell 投射到 Web 端，支持好友 URL 远程协作
    tfr key@tfr-node [cmd ...] # 通过 key 口令鉴权，远程连接 tfr-node 节点，免交互无人值守执行指定 cmd 命令
    💡 提示：若使用的是普通静态密钥（非TOTP动态口令），需先执行 tfr -n key@tfr-node [secret] 为该节点密钥预置绑定 TOTP 密钥种子

    🔗 无人值守模式（tfr -n）：

    一次极简后台服务安装，生成带 TOTP/WebAuthn 认证的固定网址 任何地方通过认证后即可获得 Web Shell，无需手动执行 tfr -t。

    • 安装与初始化：在目标节点执行 tfr -n 生成固定网址。首次访问该网址时，浏览器提示设置 TOTP 密码。
    • 节点 URL 路径语法（核心）：
      • 默认（根路径） —— https://tfr.sh.cn/p/xxxx/ → 启动 bash
      • SSH 远程 —— /root@IP/[port] → 连接指定服务器（端口可选，默认为22）; 预设其密码: tfr -n root@IP [password]
      • 容器内 Shell —— /container_id@bash 或 /container_id@sh → 进入指定容器的 bash 或 sh
      • 自定义指令 —— 可直接作为执行的命令，参数以 / 分隔（例如 /python/--version 执行 python --version
    • 细粒度认证：在上述任意路径后继续追加 /Auth可为该特定路径设置独立的认证（例如 /root@192.168.0.2/Auth）

    ⚖️ 技术中立与合规声明

    1. 原理中立：本系统基于已申请的国家发明专利架构，核心采用 Rust 异步内存流式透传技术。服务端仅充当瞬时物理管道（Blind Relay），不具备对传输数据（包括文件名、文件内容及目录结构）进行解析、重组或恢复的技术能力。

    2. 端到端封装：所有数据均在前端（浏览器/终端）完成流式封装，服务端对传输载荷表现为完全黑盒状态。系统严格执行“不落地、不存储、不审查”原则，确保传输过程的极致隐私与安全。

    3. 合规溯源：作为技术服务提供方，本系统依法履行网络安全监管义务，仅在后台留存必要的传输行为元数据（如：会话ID、双端脱敏IP、流量指纹及时间戳），日志留存期符合国家相关法律规定。

    * 提醒：请在法律允许的范围内使用本工具。用户利用本系统发起的所有传输行为，均由用户自行承担相应的法律责任。
