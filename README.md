# TFR \(Terminal File Relay\) \| 流盲终端

**✨ Rust 自研 · 无端口/无客户端/无隧道 跨网加密运维传输工具**

TFR 是基于 Rust 异步内存流式架构开发的**新一代内网互通与远程运维工具**，坐拥9项国家发明专利。彻底替代传统 VPN、SSHD，实现 DNS 隐秘触发、无端口监听、内存无痕传输、跨网段 P2P 直通，适配容器/服务器/企业内网全场景，兼顾极致安全与轻量化运维体验。

**🌐 官方站点：https://tfr.sh.cn**

---

## 📌 核心特性

- **零安装零配置**：无需 SSH 配置、无需客户端、无需虚拟网卡，一行 curl 一键部署启用

- **内存原生传输**：分块压缩流水线传输，全程零磁盘占用，跑满带宽，高效低耗

- **阅后即焚安全机制**：服务端仅作为加密盲中继管道，无法解析、存储、复原用户数据，零文件持久化

- **隐秘防拦截通信**：DNS 触发式无端口连接，规避端口扫描、防火墙拦截与 NAT 限制

- **端到端全加密**：前端完成数据封装加密，服务端黑盒转发，严格遵循不落地、不存储、不审查原则

- **容器深度适配**：容器运维体验对标虚拟机，极低内存占用，无缝替代传统 VM 运维模式

- **全自动断点续传**：终端、浏览器双向支持大文件断点续传，无需复杂命令，传输稳定不丢包

## 🛠️ CLI 完整使用手册

```bash
# 无参数：默认拉取个人传输列表选中文件
tfr

# 无人值守模式：检测/安装后台节点服务，生成固定运维链接
tfr -n

# 上传文件/目录（目录自动压缩，小文件支持在线预览编辑）
tfr [file/dir]

# 断点续传下载（需搭配发送端 tfr [file] -p 使用）
tfr [file] -c

# 标准输入流式传输（自定义文件名，默认 stdin.log）
echo "data" | tfr [?filename]

# 终端投屏：本地/容器 Shell 投射至网页，支持好友协作链接
tfr -t [?container] [?sh] [?Friend_URL]

# 密钥认证：远程操控无人值守 TFR 节点执行命令
tfr key@tfr-node [cmd ...]

```

**💡 小贴士**：浏览器端文件传输自动开启断点续传，无需额外执行命令。

## 🔗 无人值守运维模式（核心功能）

执行 `tfr -n` 完成一次性轻量化后台服务部署，生成**永久固定运维 URL**，内置 TOTP/WebAuthn 双重安全认证。首次访问设置密码后，可随时随地网页直连 WebShell，无需重复执行投屏命令。

### 节点核心路由语法

- **默认终端**：`https://tfr.sh.cn/p/xxxx/` 自动启动 bash 终端

- **SSH 远程连接**：`/root@IP/[port]` 连接指定服务器（端口默认22），支持预存账号密码

- **容器终端进入**：`/container_id@bash` 或 `/container_id@sh` 进入指定容器终端

- **自定义命令执行**：路径参数化执行命令，例 `/python/--version` 执行 python --version

- **细粒度独立认证**：任意路径后缀添加 `/Auth`，为单一路径单独加密防护
